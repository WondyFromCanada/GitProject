git이란? - git을 사용하는 이유 /20191231

- 버전 관리 시스템인 git을 사용함으로써 파일 변화를 시간에 따라 기록했다가 나중에
특정 시점의 버전을 다시 꺼내올 수 있다.
- 분산 버전 관리 시스템(DVCS)로 동시에 다양한 그룹과 다양한 방법으로 협업할 수
있다.
- 계층 모델 같은 중앙집중식 시스템으로는 할 수 없는 워크플로를 다양하게 사용할 수
있다.

프로젝트 3단계

1.Working directory란

- 실제 작업이 일어나는 곳
- 최초 git directory에서 checkout을 받아 작업하게 되는 공간
- Staging area로 Stage Fixes

2.Staging Area란

- 추가되거나 변경된 사항의 리스트가 저장되는 공간
- 커밋을 하면 고유 번호 (SHA-1 해시)할당됨
- git directory로 Commit

3.Git directory란

- git이 프로젝트의 메타데이터와 객체 데이터베이스를 저장하는 곳
- Working Directory로 Checkout the project

git 최초 설정

- 사용자 정보

$ git config --global user.name "John Doe“
$ git config --global user.email johndoe@example.com

- For Windows

* 편집기 설정
$ git config --global core.editor "'/c/Program Files (x86)
/Notepad++/notepad++.exe' -multiInst -notabbar
-nosession -noPlugin“

* 자동 줄바꿈 설정
$ git config --global core.autocrlf true

* 원격 저장소와 로컬 저장소 동기화
$ git config -- global fetch.prune true

명령어

- 설정이 잘 되었는지 확인

$ git config --global --list

- Git 내부명령어 확인

$ git

- Git 저장소 만들기

* 기존 디렉토리를 Git 저장소로 만들기 : $ git init

* 기존 저장소를 Clone 하기 : $ git clone https://github.com/
WondyFromCanada/gitProject.git

- 저장소에 저장하기

* 파일 상태 확인하기 : $ git status

* Modified 상태의 파일을 Stage 하기: $ git add.

* 변경사항 커밋하기 : $ git commit

- 커밋 히스토리 조회하기

* $ git log
* $ git log --decorate --graph --all --oneline

- 리모트 저장소

* 리모트 저장소 추가하기 : $ git remote add origin
https://github.com/WondyFromCanada/gitProject.gitProject.git

* 리모트 저장소에 Push 하기 : $ git push --set --upstream origin master

- 파일 무시하기

* $ gitignore

- 리모트 저장소에서 가지고 오기

* $ git pull

명령어2 /20200102

- 커밋 되돌리기

1.commit을 취소하고 해당 파일들은 staged 상태로 워킹 디렉터리에 보존
- $ git reset --soft HEAD^

2.commit을 취소하고 해당 파일들은 unstaged 상태로 워킹 디렉터리에 보존
- $ git reset --mixed HEAD^
- $ git reset HEAD^
- $ git reset HEAD~2

3.commit을 취소하고 해당 파일들은 unstaged 상태로 위킹 데릭터리에서 삭제
- $ git reset --hard HEAD^

- 커밋 메시지 변경하기

* $ git commit --amend

- 리모트 저장소 -> 새로 만듦

* 리모트 저장소 추가하기 : $ git remote add origin
https://github.com/WondyFromCanada/GitProject.git

* 리모트 저장소에 Push 하기 : $ git push --set-upstream origin master

- 리모트 저장소에서 가지고 오기

* $ git fetch origin : 원격 저장소에서 데이터를 로컬로 가져옴

* $ git merge origin/master : 원격 저장소의 master 브랜치를 병합

* $ git pull : 원격 저장소의 내용을 가져와 자동으로 병합

Git 브랜치

- Git 브랜치

* 현재 로컬에 존재하는 브랜치 확인하기 : $ git branch

* 새로운 브랜치 생성하기 : $ git branch 브랜치명

* 브랜치 이동하기 : $ git checkout 브랜치명

* 브랜치 생성과 동시에 이동하기 : $ git checkout -b 브랜치명

* 브랜치 삭제 : $ git branch -d 브랜치명

Merge의 기초

- Merge

* $ git merge 브랜치명 : 합칠 브랜치에서 합쳐질 브랜치를 merge

- Merge 할 때 발생하는 상황

1.Fast-Forward

2.Tree-way Merge

- Conflict

* 충돌 : Merge하려는 두 브랜치에서 같은 파일의 같은 부분을 수정하고 merge할 때 충돌 발생

* 누가 충돌을 발생시켰는지 확인하고 싶을 때 : $ git blame 파일명

Tag

- 태그

* 태그 조회하기 : $ git tag

- 태그 붙이기

1. Lightweight
* $ git tag 태그명 -> 특정 커밋에 대한 포인터

2. Annotated
* $ git tag -a 태그명
-> 태그를 만든 사람 이름, 이메일, 태그를 만든 날짜, 태그 메시지 저장

- 태그 정보 확인하기

* $ git show 태그명
• Lightweight : 커밋 정보만 확인 가능 • Annotated : 태그 정보와 커밋 정보 모두 확인 가능

- 태그 공유하기

※ 태그를 만들면 서버에 별도로 Push해야 함 (자동으로 리모트 서버에 태그를 전송하지 않음)
* $ git push origin 태그명 -> 태그를 이용하여 checkout도 가능
* $ git push origin --tag

Stashing

- Stash

* 수정하던 것을 스택에 저장 : $ git stash 또는 $ git stash save 이름

* 저장한 Stash 목록 확인하기 : $ git stash list

* 저장한 Stash 불러오기 : $ git stash apply 이름 또는 $ git stash drop 아이디

* apply와 drop을 통합하는 $ git stash pop
